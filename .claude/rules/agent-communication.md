# Agent communication & approval rules

Shared discipline for `home` and every money-loop engine (`pipeline`, `audit`, `delivery`,
`retain`). Mirrors the total-audio-platform agent-communication rules, adapted to chrisscho.uk's
draft-only consulting context.

## Ground every claim

Audit each progress claim against a tool result from **this run** before reporting it. Only report
work you can point to evidence for. If something is unverified, say so. If a check or a send-draft
fails, report it with the output — never smooth it over. No invented client counts, testimonials, or
borrowed credibility.

## Write the final report for a reader who saw none of the run — and keep it selective

Outcome first, in one plain sentence. Then only the detail that changes what Chris does next. Then
the one or two things you need from him. Complete sentences — no arrow chains or shorthand invented
mid-run. Files, drafts, prospects, and client names each get their own plain clause. Get concise by
including less, not by compressing the writing.

## Pause only where the work genuinely needs Chris

Draft-only is absolute: **every client-facing message, report, SOW, post, invoice, or sequence is
drafted for Chris to review and send.** Nothing is sent, scheduled, armed, or auto-fired.

Pause and hand back at a **real gate**:

- a send (email, DM, outreach, client comms)
- a spend or a price change the client hasn't agreed
- a booking or a calendar commitment
- an irreversible or destructive action
- a genuine scope change
- input only Chris can provide (a decision, a credential, a judgement call on tool fit)

Ask, then **end the turn** — don't end on a promise to continue. Everything reversible that follows
directly from the request: proceed without asking.

## House style

GBP, UK spelling, calm/professional tone, name-only greeting in client copy ("Sarah,"), no emojis,
no hype. Client data lives only in `.claude/clients/` (git-ignored) — never commit it.
