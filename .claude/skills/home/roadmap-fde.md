# Roadmap — up-market to FDE (Forward-Deployed Engineer)

Where chrisscho.uk is heading *above* the current SMB audit business. This is a **growth vector, not a
current operating mode** — the engines run the £750-audit money loop today; FDE is the premium tier
they grow into once the capability gate is cleared. Source: Chris's FDE 30-Day Sprint note
(2026-07-16), triggered by Greg Isenberg's FDE thread.

## The move in one line

From **SMB audits** (owner-run 2–20-person businesses, £750 wedge → £2–8k builds → £900–1,500/mo
concierge) to **FDE engagements** (embedded inside larger companies, enterprise stakeholders,
£5k+ projects / £4k+/mo) — a higher ICP at a higher price for the same underlying skill: deploying AI
systems that work.

## It's gated — don't sell or build FDE before the gate clears

The up-market move is unlocked by the **30-Day Sprint** (tracked in Linear: `fde-30-day-plan`,
TOT-119…128), not available yet. The gate:
- Anthropic Academy certs (Claude Code 101, Platform 101, Building with the Claude API, MCP).
- Production Python confidence (the current biggest gap — async, uv, testing, packaging).
- RAG + evals methodology (structured eval, trace analysis, metrics).
- **One shipped Python project** as the proof point (e.g. RAG vault search, or a client analytics
  dashboard on the PostHog API).
- **One FDE case study** (the Newsjack daemon is a ready-made story: automated feed monitoring → AI
  grading → drafting in Chris's voice).

Until those exist, chasing £5k+ enterprise work just burns focus — the same trap the concierge doc
flags about paid acquisition. **The proven SMB audit loop is the cash-now base; protect its focus.**

## How FDE maps onto the existing engine (no new engine needed)

| Engine | FDE role |
| --- | --- |
| `audit` | Stays the wedge. For a larger client the £750 audit is paid discovery that *escalates* into an FDE engagement — same blueprint, bigger next step. |
| `delivery` | The FDE build **is** the top of the delivery ladder — a bigger, embedded implementation at £5k+. |
| `retain` | The FDE retainer **is** the "rung above the concierge" already noted in `../music-ai-audit/upsells-and-concierge.md` (£4k+/mo, fractional-AI-lead). |
| `retain` → `case-study` | Week 4 of the sprint runs straight through the case-study blueprint. |
| `pipeline` | Enterprise/FDE prospects are a **future Track C** — a distinct ICP pool, kept separate from Track A (music) and Track B (Brighton local) under the pool-separation rule. Do not start Track C prospecting until the gate clears. |

## What the engines should do *now* about FDE

- **Recognise the signal, don't chase it.** If a request or a prospect is clearly FDE-sized (a larger
  company, an embedded/enterprise ask, a £5k+ opportunity), **flag it to Chris as the premium path** —
  don't quietly shrink it into a £750 audit, and don't spin up enterprise delivery machinery.
- **Feed the gate.** Anything that produces an FDE proof point (a shipped Python system, a strong case
  study) is high-value — surface it.
- **Otherwise, carry on.** Keep running the SMB loop. Revisit this roadmap when the sprint completes;
  at that point the likely additions are a Track C prospecting profile and an explicit FDE tier on the
  offer page.
